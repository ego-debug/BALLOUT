# Ballout

A Roblox dodgeball game. Setup + workflow notes so this project can be picked
up on any machine.

---

## What's in here

```
BALLOUT.rbxl            ← THE LIVE PLACE FILE. The HQ map lives inside it.
src/                    ← All game code (synced into Studio by Rojo)
  Shared/BallConfig.luau  ← every feel/tuning number lives here
  Server/                 ← ball, animation, ballform, glider, bot services
  Client/                 ← camera, input, movement, HUD
  Character/              ← custom Animate script
tools/build_map.luau    ← regenerates the whole HQ map (see below)
default.project.json    ← Rojo mapping (which folder goes where in Studio)
DESIGN.md               ← full game design spec
CLAUDE.md               ← project context, auto-loaded by Claude Code
```

`Place1.rbxl` is a **stale** copy from 2026-07-13 (before the map existed).
Kept only as a fallback. **Do not open it** — open `BALLOUT.rbxl`.

## ⚠️ The one thing that can lose work

This project has **two halves that sync differently**:

| | Where it lives | How it moves between machines |
|---|---|---|
| **Code** | `src/*.luau` (text) | git — merges fine |
| **Map / HQ geometry** | inside `BALLOUT.rbxl` (binary) | git, but **cannot merge** |

`.rbxl` is a binary file. If you edit the map on the desktop *and* on the
laptop without syncing in between, git **cannot merge them** — you have to throw
one away.

### Golden rule

> **`git pull` before you start. `git push` when you stop. Every time.**

If you only ever work on one machine at a time and follow that rule, nothing can
be lost.

## First-time setup on a new machine

### 1. Install the tools

- **Git** — https://git-scm.com/download/win (accept the defaults)
- **Roblox Studio** — sign in with the same Roblox account
- **Rojo 7.7.0** — download `rojo.exe` from
  https://github.com/rojo-rbx/rojo/releases and put it somewhere permanent,
  e.g. `C:\Tools\Rojo\rojo.exe`
- **Claude Code** — however you installed it on the desktop

### 2. Get the project

```bash
git clone <your-repo-url> "BALLOUT FILE"
cd "BALLOUT FILE"
```

### 3. Install the Rojo Studio plugin

In Studio: **Plugins → Manage Plugins → Marketplace**, search **"Rojo"**,
install it. (Or run `rojo plugin install` from the project folder.)

### 4. Install the Roblox Studio MCP plugin

This is what lets Claude build things in Studio directly, instead of only
writing code. Install the Roblox Studio MCP plugin, then in Studio open
**File → Studio Settings → Assistant** (or the plugin's own toggle) and enable
**"Enable Studio as MCP server."**

Then **restart Studio**. The MCP connects to **whichever Studio window is
active** — if you have more than one place open, it can grab the wrong one.

### 5. Start working

```bash
# From the project folder:
"C:\Tools\Rojo\rojo.exe" serve --port 34872
```

Open `BALLOUT.rbxl` in Studio → open the Rojo plugin → **Connect**.

## Daily workflow

```bash
git pull                    # ALWAYS first
# ... work ...
git add -A
git commit -m "what changed"
git push                    # ALWAYS last
```

**In Studio**, the sync loop that avoids stale-code bugs:

```
stop play  →  wait ~6 seconds for Rojo to sync  →  start play
```

Rojo **only syncs while Studio is in Edit mode**. Starting a play session too
fast runs the *old* code — this has caused more phantom bugs in this project
than any real one.

**After changing the map, press Ctrl+S in Studio**, then commit `BALLOUT.rbxl`.
Rojo does *not* carry map changes.

## Changing the map

The HQ is generated by a script, not hand-built:

1. Edit `tools/build_map.luau`
2. Run it in Studio in **Edit mode** (Claude can do this via `execute_luau`)
3. It destroys and rebuilds `Workspace.BalloutMap` from scratch
4. **Ctrl+S** in Studio
5. Commit `BALLOUT.rbxl`

---

## Tuning game feel

Everything — throw speeds, catch windows, stun times, dash distance, camera,
glide, ballform — is in **`src/Shared/BallConfig.luau`**. Change numbers there
and re-sync; no need to touch the service files.
