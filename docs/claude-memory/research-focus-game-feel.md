When researching Knockout City for [[ballout-project]], focus on **what makes the game feel like the game** so it can be recreated: ball size, ball speed, player movement/run speed, jump height, gliding, dodge/roll distances, and precise timing windows for catches, charged throws, and hits. Ball types and their behaviors matter too.

**Why:** The user wants to replicate the exact feel and import it into Roblox, not just a rough approximation.

**How to apply:** Chase specific numbers/timings/inputs, not marketing fluff. The netcode question ("how to run catch/hit timing so it doesn't lag/glitch") is explicitly a LATER concern — the user wants a custom approach for that at implementation time, not during research.
